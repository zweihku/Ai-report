OpenClaw 协议: Agent-to-System 标准化交互层 OpenClaw

Sakana AI: Doc-to-LoRA -- 将检索增强编码进权重 Sakana AI

Sakana AI 提出 Doc-to-LoRA，核心思路是将文档信息直接编码为 LoRA 权重，实现"即时上下文内化"而绕过传统 RAG pipeline（embedding -> retrieval -> augmentation）。

技术细节
  • 将文档通过快速微调转化为低秩适配器（LoRA），推理时直接激活相应 LoRA
  • 消除了检索阶段的延迟，但引入了 LoRA 容量瓶颈 -- 单个 LoRA 的参数量限制了可"记忆"的信息量
  • 多文档场景需要 LoRA 合并或路由策略（类似 MoE 的 expert routing）
  • 与 RAG 相比，更新文档需要重新训练 LoRA，不适合频繁变化的知识库
研究意义：这项工作模糊了 retrieval-augmented generation 和 parameter-efficient fine-tuning 的边界，为"知识内化"提供了新范式。局限在于 LoRA 的有限容量和更新成本。

ICML LLM 审稿检测与学术诚信 学术

ICML 对使用 LLM 审稿的审稿人采取了论文拒收处分。检测 LLM 辅助审稿的技术挑战：

  • 审稿文本短（通常 500-2000 词），传统 AI 文本检测器在短文本上 F1 显著下降
  • 领域术语密集，容易产生误报（人类专家用语可能与 LLM 输出相似）
  • 检测手段可能包括：stylometric 分析、审稿时间戳异常、跨审稿人一致性检查
  • 根本问题：LLM 辅助与 LLM 替代的边界模糊（用 LLM 润色 vs 用 LLM 直接生成评审意见）

Reddit r/MachineLearning 179 upvotes / 69 comments。另一个热帖："ICLR 2026 oral with 2 rejects, 1 borderline reject"（82 upvotes）反映了学术审稿一致性问题。
