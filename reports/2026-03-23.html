模型更新
Gemini 3.1 Pro：多模态推理的新标杆 模型

Google DeepMind 发布 Gemini 3.1 Pro，在 ARC-AGI-2 测试中达到 77.1% 的准确率，在 AI Analysis Index 上与 GPT-5.4 Pro 并列 57 分。模型原生支持文本、图像、音频、视频和代码五种模态的联合推理。

Gemini 3.1 Pro 的核心技术进步在于统一的多模态注意力机制，不再需要为不同模态设置独立的编码器管线，而是在 Transformer 内部直接对齐多种模态的表示空间。

同步推出的 Flash-Lite 变体将推理延迟降低 2.5 倍，输出速度提升 45%，API 定价 $0.25/M 输入 token，适合大规模部署场景。

MiMo-V2-Omni：小米的全模态 Agent 模型 模型

小米发布 MiMo-V2-Omni，定位为全模态 Agent 模型，能处理文本、图像和视频输入，输出文本和自然语音（通过 MiMo-V2-TTS）。此前传闻的 "DeepSeek V4 Hunter Alpha" 实际为小米的 MiMo-V2-Pro。

技术亮点在于 TTS 模块采用了基于 diffusion 的声码器，支持 zero-shot 声音克隆和情感控制，语音自然度显著优于传统 TTS 方案。

NVIDIA Nemotron 3 Super：MoE 架构的效率突破 模型

NVIDIA 在 GTC 2026 上发布的 Nemotron 3 Super 采用 120B 参数的 Mamba-Transformer 混合 MoE 架构，活跃参数仅 12B，支持 1M 上下文窗口，吞吐量是同类开源模型的 2.2 倍。

Mamba 层处理长序列时的线性复杂度与 Transformer 的注意力机制互补，实现了高效长文本推理。模型已开源，适用于高吞吐 Agent 场景。

Kimi K2.5 开源："群体模式"可调度100个子 Agent 模型

Moonshot AI 开源了 Kimi K2.5 多模态模型，最大亮点是 "swarm mode"，允许主 Agent 并行调度最多 100 个子 Agent 协同完成任务。这种架构适合需要大规模并行处理的复杂工作流场景。

关键架构创新：Gemini 3.1 采用统一多模态注意力层（Unified Multimodal Attention），五种模态的 token 在同一个注意力矩阵中交互，而非传统的模态独立编码 + 后期融合。这使得跨模态推理（如"这段代码的输出图像中有什么问题"）的表现大幅提升。

Flash-Lite 变体采用更激进的量化策略（W4A8）和投机解码（speculative decoding），在保持 95% 以上质量的同时将延迟降至原版的 40%。API 定价 $0.25/M 输入 token、$1.00/M 输出 token。

NVIDIA Nemotron 3 Super：Mamba-Transformer 混合 MoE 模型

Nemotron 3 Super 的核心架构是 Mamba-Transformer 混合 MoE。120B 总参数中每次推理仅激活 12B（10% 激活率），在 1M 上下文窗口下实现了 2.2x 的吞吐量提升：

  • Mamba 层（占 60%）：处理长程依赖，O(n) 复杂度的选择性状态空间模型，高效处理超长上下文
  • Transformer 层（占 40%）：处理局部注意力和精细推理，使用 Grouped Query Attention (GQA) 降低 KV cache 内存
  • MoE 路由：Top-2 expert 选择，128 个 expert 组，使用 load-balancing auxiliary loss 防止路由坍缩
  • 训练数据：15T tokens，包含 3T 代码 token，采用三阶段课程学习（curriculum learning）

模型权重已在 NVIDIA NGC 和 Hugging Face 上开源，支持 vLLM 和 TensorRT-LLM 部署。

MiMo-V2-Omni 与 MiMo-V2-TTS 技术栈 模型

小米 MiMo-V2-Omni 采用多阶段训练策略：

  • Stage 1 - Vision Encoder Warm-up：冻结 LLM，仅训练视觉编码器在大规模图文数据上对齐
  • Stage 2 - Multimodal Instruction Tuning：全参数微调，支持文本/图像/视频输入
  • Stage 3 - Agent Alignment：通过 RLHF + tool-use 数据对齐 Agent 行为

MiMo-V2-TTS 采用 diffusion-based vocoder，支持 zero-shot voice cloning（3秒参考音频）和 8 种情感风格控制。延迟 <200ms（首 token），码率 24kHz/16bit。

确认此前在野外出现的 "DeepSeek V4 Hunter Alpha" 实为 MiMo-V2-Pro 的内部代号泄露。

Kimi K2.5 Swarm Mode 架构 模型

Moonshot AI 开源 Kimi K2.5，其 Swarm Mode 的技术实现：

  • 调度架构：主 Agent 作为 orchestrator，通过 task graph 将复杂任务分解为 DAG，分发给最多 100 个子 Agent 并行执行
  • 通信协议：基于 structured message passing，子 Agent 间可通过共享 memory pool 交换中间结果
  • 容错机制：单个子 Agent 失败时自动重试或 re-route 到备用 Agent，不影响整体任务
  • 效率提升：在复杂代码重构任务上，Swarm Mode 相比单 Agent 执行速度提升 8-15x
研究深度解析
Gemini Robotics 1.5 / Robotics-ER 1.5：端到端 VLA 模型 研究

Google DeepMind 的 Gemini Robotics 1.5 是首个将"思考型"（thinking）推理能力引入 Vision-Language-Action (VLA) 模型的工作：

  • VLA 统一架构：视觉输入 -> 语言推理 -> 连续动作空间输出，全部在一个 Transformer 内完成。动作输出使用 diffusion head 生成连续控制信号
  • Embodied Reasoning（ER 变体）：增加了"内在思维链"，机器人在执行前生成中间推理步骤（如"桌上有3个物体，用户要求整理，应该先按类别分组"），然后才输出动作序列
  • 训练数据：混合了 Internet-scale 视觉-语言数据和 robot manipulation 数据（来自 Google 的 RT-X 数据集），采用分阶段训练
  • Benchmark：在 RLBench 和 CALVIN 上刷新 SOTA，尤其在需要 multi-step reasoning 的任务上提升显著（+23% vs RT-2X）
DeepMind Blog - Gemini Robotics 1.5
微软 Argos：Agentic Verification for Multimodal RL 研究

Argos 的核心贡献是将 agentic verification 引入多模态强化学习：

  • Verifier Agent：独立训练的验证模型，在主模型每个推理步骤后检查逻辑一致性和事实准确性
  • Reward Shaping：将验证结果作为辅助 reward signal 注入 RL 训练循环，引导主模型学习更可靠的推理路径
  • 结果：视觉问答幻觉率降低 37%，空间推理准确率提升 22%，推理延迟仅增加 8%（因为验证和生成可部分并行）
  • 与 Self-Consistency 的区别：Self-Consistency 通过多次采样取多数投票，而 Argos 的 Verifier 是结构化检查，能指出具体错误位置并提供修正建议

Jetson AGX Thor + mmWave 3D 感知融合 研究

TI 与 NVIDIA 展示的 Jetson Thor + mmWave 集成方案技术要点：

  • 传感器融合策略：Early fusion -- mmWave 点云和 RGB 图像在特征提取阶段就进行对齐和融合，而非传统的后期融合
  • mmWave 优势：60GHz/77GHz 毫米波雷达不受光照、雾霾、灰尘影响，测量精度达厘米级，特别适合工厂和仓库等恶劣环境
  • 延迟指标：端到端感知-决策延迟 <15ms，满足实时机器人控制需求
  • 与 Gemini Robotics 的协同：Thor 提供边缘推理算力，Gemini Robotics 提供高层规划能力，两者可组成分层控制架构
