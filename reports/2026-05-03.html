Musk v. Altman 审判第一周：xAI 承认蒸馏 OpenAI 模型 焦点

Musk 与 OpenAI 的世纪审判在奥克兰联邦法院完成了第一周。Musk 连续三天出庭作证，声称自己被 Altman 和 Brockman"欺骗"，把捐给非营利组织的 $3800 万变成了商业公司的启动资金。他还在法庭上警告"AI 可能杀死我们所有人"。

但最大的爆点来自 Musk 自己：在交叉质证中，他承认自己创办的 xAI 一直在使用"蒸馏"技术从 OpenAI 模型中提取知识。法官当场指出了矛盾 -- Musk 一边起诉 OpenAI 背叛开源使命，一边自己的公司却在利用 OpenAI 的模型输出来训练。审判预计持续到 5 月 21 日，Altman 和 Brockman 将在本月晚些时候出庭。

Roze 将整合 SoftBank 旗下的能源、土地和基础设施资产，以及去年收购的全球机器人巨头 ABB Robotics。SoftBank 计划 7 月在德州数据中心举办分析师日为 IPO 造势，KPMG 已经被聘请准备财务文件。这是 AI 基础设施投资从"算力军备竞赛"演进到"自动化建设竞赛"的标志。

这对企业意味着什么？原来很多公司都指望着改革能把 2026 年 8 月的合规截止日推迟到 2027 年底，现在这个假设崩塌了。如果下个月的谈判再次失败，全球做欧洲生意的 AI 公司只剩大约 3 个月来确认自己的合规准备是否到位。

焦点事件
Musk v. Altman 审判：模型蒸馏的法律与伦理灰色地带 焦点

审判第一周最具技术影响力的披露是 Musk 承认 xAI 使用了"蒸馏"（distillation）技术从 OpenAI 模型中提取知识。模型蒸馏是将大模型（teacher）的输出用于训练小模型（student）的标准 ML 技术，但在商业竞争中使用竞争对手的 API 输出进行蒸馏，触及了 API 服务条款和知识产权的灰色地带。

OpenAI 的 API 使用条款明确禁止使用模型输出来训练竞争性 AI 系统。xAI 的蒸馏行为如果被证实系统性地违反了这些条款，可能引发独立的知识产权诉讼。这一案例可能为 AI 行业的"模型间知识传递"设定法律先例。

从技术角度看，蒸馏已经成为行业普遍做法 -- DeepSeek 在开源模型训练中也被指控使用了类似技术。但法律框架尚未跟上：现有的知识产权法主要保护训练数据的版权，而非模型输出的"知识"。审判的裁决可能重塑整个行业的模型训练规范。

政策与监管
EU AI Act Omnibus 谈判破裂：Annex III 合规窗口收窄 政策

欧盟议会与理事会就 Digital Omnibus for AI 的三方谈判（trilogue）在 4 月 28-29 日经过 12 小时的马拉松会议后破裂。核心分歧在于议会希望将 Annex I 中的部分法规从 Section A 移至 Section B -- 一个看似程序性的调整，但实质影响了哪些行业特定法规优先于 AI Act。

如果下月谈判再次失败，Annex III 高风险 AI 系统的合规截止日将维持在 2026 年 8 月 2 日，而非此前预期的推迟至 2027 年 12 月。这意味着全球向欧盟出口 AI 产品的公司只剩约 3 个月的合规准备时间。

对于使用 AI API 构建产品的开发者，关键影响点：高风险分类下的系统（医疗诊断、信用评分、招聘筛选等）需要完成合规性评估（conformity assessment）、建立风险管理系统、保存技术文档。如果依赖 Omnibus 延期的假设做规划，现在需要启动应急方案。

NVIDIA Nemotron 3 Nano Omni：9 倍吞吐量的多模态 Agent 模型 模型

NVIDIA 发布 Nemotron 3 Nano Omni，一个开源多模态模型，支持视频、音频、图像和文本的统一推理。核心卖点是在同等交互质量下实现 9 倍于其他同类开源全模态模型的吞吐量，显著降低了 Agent 应用的推理成本。

该模型专为 Agent 工作流设计 -- 能同时处理视觉输入（屏幕截图、实时视频流）和语音指令，适用于需要多感官理解的自动化场景。配合 NVIDIA 的 Isaac GR00T 机器人模型和 Cosmos 世界模型，形成了从感知到决策到执行的完整 Physical AI 技术栈。

焦点事件
模型蒸馏的法律前沿：xAI v. OpenAI ToS 合规性分析 焦点

Musk v. Altman 审判第一周（4/28-5/2）中，Musk 在交叉质证中承认 xAI 使用了知识蒸馏（knowledge distillation）技术从 OpenAI 的模型输出中训练自己的模型。这是业界首次在法律程序中公开确认这种做法。

技术细节：蒸馏的实施路径

标准知识蒸馏流程：通过 API 向 teacher model（此处为 GPT 系列）发送大量 prompt，收集输出的 logits 或 token 序列，用这些数据作为 soft labels 训练 student model。OpenAI API ToS 第 2(c) 条明确禁止"使用输出开发与 OpenAI 竞争的模型或服务"。但执行层面的难点在于：如何区分"学习 API 行为用于产品集成"和"系统性蒸馏用于模型训练"。

法律影响面
  • API 输出的版权归属：根据美国版权局 2023 年指南，纯 AI 生成的内容不受版权保护。但如果 prompt 设计包含实质性人类创造性输入，输出可能受到保护
  • 合同法 vs. 知识产权法：ToS 违约是合同法问题，赔偿通常限于实际损失；而如果 OpenAI 能主张输出的"汇编版权"（compilation copyright），则可能适用版权法的法定赔偿
  • 行业先例效应：DeepSeek、Mistral 等开源模型训练中也被指控使用了类似蒸馏技术。本案裁决可能为整个行业设定基准

MIT Technology Review
SoftBank Roze 技术栈：ABB Robotics + Isaac GR00T + Cosmos 世界模型 行业

Roze 的技术架构将整合三层能力：ABB Robotics 的工业机器人硬件（IRB 系列关节臂、GoFa 协作机器人）、NVIDIA Isaac GR00T 的视觉语言动作（VLA）模型用于任务理解和规划、以及 Cosmos 世界模型用于环境建模和合成数据生成。

数据中心自动化的技术挑战
  • 结构化环境优势：数据中心的建设环境比一般建筑工地更加标准化（重复性机架安装、线缆布设、冷却管道），适合机器人自动化
  • Sim-to-Real 转移：NVIDIA Cosmos 世界模型可以生成大量合成训练数据，减少真实环境中的数据采集成本。Isaac GR00T 的多步骤任务理解能力使机器人能处理序列化工作流
  • 关键瓶颈：电气接线和精密设备调试仍需人工干预，初期自动化率预计在 60-70% 之间
市场定位

当前数据中心建设面临约 40% 的劳动力缺口，平均建设周期为 18-24 个月。Roze 如果能将自动化率提升至 70% 并将建设周期缩短 30%，在 $3000 亿+ 的年度数据中心建设市场中将占据显著份额。$1000 亿估值对应约 33x 预期收入倍数 -- 如果 Roze 能在首年实现 $30 亿+ 收入。

Apple Foundation Models 架构：Gemini-based 训练 + PCC 推理的混合部署 产品

Apple WWDC 2026 将揭示 iOS 27 中 Siri 的重大架构变更。底层技术基于 Apple 与 Google 的多年合作 -- Apple Foundation Models (AFM) 的训练基于 Gemini 架构，但推理分为两条路径：

  • 设备端推理：较小的 AFM 模型在 A20/M5 芯片上运行，处理低延迟任务（快速回答、屏幕上下文理解、应用内操作）
  • Private Cloud Compute (PCC)：较大的 AFM 模型在 Apple 自有的 Apple Silicon 服务器上运行，处理复杂任务。PCC 的关键差异化：代码和数据在 Secure Enclave 中处理，Apple 无法访问用户数据
Extensions 架构的技术实现

iOS 27 的 Extensions 系统允许第三方 LLM（Claude、Gemini、Grok 等）注册为 Siri 查询的替代后端。技术上，这是一个 intent routing 层 -- Siri 首先用本地 NLU 模型理解查询意图，然后根据用户的 Extension 偏好决定将查询路由到内置 AFM 还是第三方 LLM API。

对 LLM 提供商的影响：Extensions 模式下，LLM 的调用成本由用户承担（通过各提供商的订阅），Apple 不收取中间费用 -- 但获得了用户偏好数据和查询模式洞察。这可能是 Apple 在不直接参与 LLM 竞争的情况下，通过分发渠道控制力影响市场格局的战略选择。
