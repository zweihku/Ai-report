和你有关的事
AI 现在能帮你写歌了，还挺像样 生活

Google 发布了 Lyria 3 Pro，这是一个能生成长达 3 分钟完整歌曲的 AI 模型。它可以创作包含人声、副歌、主歌的完整曲目，而且听起来还挺专业。已经在 Gemini App 上向付费用户开放了。

如果你喜欢音乐，可以去试试。它不会取代真正的音乐人，但确实能帮普通人快速做出一首还不错的背景音乐或者玩玩创意。对专业音乐人来说，这更像是一个新的创作工具，而非威胁。
GitHub 要拿你的代码来训练 AI 了 注意

从 4 月 24 日起，GitHub 将默认使用 Copilot 免费版、Pro 和 Pro+ 用户的交互数据（包括你输入的代码和 AI 的回复）来训练 AI 模型。不过你可以在隐私设置里选择退出。

如果你用 GitHub Copilot，建议去设置里看一下，决定要不要让他们用你的代码数据。企业版用户不受影响。这件事的本质是：你用免费工具，代价就是你的数据。
AI 要帮你网购了，而且是真的帮你下单 生活

Shopify 和 Google 联合推出了"通用商业协议"（UCP），让 AI 助手可以在 ChatGPT、Google 搜索、Gemini 里直接帮你找商品、比价、下单。沃尔玛、Target、Etsy 等大品牌都加入了。

想象一下对 AI 说"帮我买一双跑步鞋，预算 500 以内，要透气的"，然后它帮你找好、比好价、直接下单。这个未来正在变成现实，但离普及还需要一段时间。

Google Lyria 3 Pro：AI 音乐生成的新标杆 模型

Google DeepMind 发布 Lyria 3 Pro，在 Lyria 3 基础上进一步提升，支持生成长达 3 分钟的完整曲目，具备专业级结构感知能力。模型支持精细控制，包括节拍条件设定、时间对齐歌词、以及多模态输入（可通过图片影响曲风）。

技术点评：Lyria 3 Pro 的核心突破在于"结构连贯性"--能从第一个音符到最后一个音符维持音乐逻辑一致性，这在此前的音乐生成模型中是主要痛点。通过 Gemini API 和 AI Studio 向开发者提供公开预览，意味着音乐类应用的开发门槛大幅降低。目前已覆盖 Gemini App、Google Vids、ProducerAI、Vertex AI 等 6 个平台。

GitHub Copilot 数据政策变更引争议 政策

GitHub 宣布自 4 月 24 日起，Copilot Free、Pro 和 Pro+ 用户的交互数据（输入、输出、代码片段及上下文）将默认用于 AI 模型训练。用户可通过隐私设置选择退出，此前已设置的退出偏好将被保留。Enterprise 和 Business 用户不受影响。

模型与技术深度
Google DeepMind Lyria 3 Pro：音乐生成架构分析 模型

Lyria 3 Pro 是 Google DeepMind 在 Lyria 3 发布一个月后推出的旗舰音乐生成模型，定位于"全长歌曲生成的行业标准"。模型具备以下关键技术特性：

  • 生成长度：最长约 3 分钟完整曲目，含人声、verse、chorus 结构
  • 结构连贯性：从首音到尾音维持音乐逻辑一致性，这是此前模型（如 Suno、Udio）的主要短板
  • 精细控制：Tempo Conditioning（精确 BPM 设定）、Time-Aligned Lyrics（歌词时间轴对齐）
  • 多模态输入：支持图像输入影响曲风、情绪和氛围
  • API 接入：通过 Gemini API 和 AI Studio 开放 public preview，支持 REST 调用
  • 平台覆盖：Gemini App、Google Vids、ProducerAI、Vertex AI、Gemini API、AI Studio
深度分析：Lyria 3 Pro 的发布策略值得关注--Google 选择了"平台化"路径，同时向消费者（Gemini App）和开发者（API）开放。相比 Suno 和 Udio 的独立应用模式，Google 的嵌入式策略可能更具长期竞争力。技术层面，Time-Aligned Lyrics 暗示模型内部维护了一个时间轴对齐的多模态表示，这与视频生成模型的时序建模方法有共通之处。
Google DeepMind 官方
Universal Commerce Protocol (UCP) 技术架构解析 产品

Shopify 与 Google 联合发布的 UCP 是面向 AI 代理的电商交互标准，已获 20+ 零售商和平台支持。以下为核心技术架构：

  • 分层架构：Shopping Service（交易原语如 checkout session、line items）-> Capabilities（Checkout、Orders、Catalog，独立版本化）-> Extensions（域特定 schema 扩展）
  • 协议兼容：支持 REST、MCP (Model Context Protocol)、AP2 (Agent Payments Protocol)、A2A (Agent-to-Agent)
  • 入口清单：ucp.json 作为 AI 代理的服务发现文件，类比 robots.txt 但面向 AI agent
  • 结账代理：AI agent 可代表用户完成完整结账流程，适配任意商业技术栈
  • 合作方：Walmart、Target、Etsy、Wayfair、American Express、Mastercard、Stripe、Visa
深度分析：UCP 的出现标志着 Agentic Commerce 从概念走向标准化。几个关键技术决策值得关注：(1) 选择组合式架构而非单一 API，降低了集成门槛；(2) 同时支持 MCP 和 A2A 说明 Google 在押注多协议共存而非赢者通吃；(3) ucp.json 可能成为继 robots.txt、sitemap.xml 之后第三个 Web 基础设施级别的标准文件。Shopify 报告 AI 驱动的购物订单自 2025 年 1 月以来增长了 15 倍，验证了市场需求。
Shopify Engineering Blog
OpenAI Safety Bug Bounty：Agentic AI 安全范式转变 安全

OpenAI 推出的 Safety Bug Bounty 计划标志着 AI 安全从"模型安全"向"代理安全"的范式转变。与传统 Security Bug Bounty 的区别：

  • 接受范围扩大：不要求构成传统安全漏洞，只需证明"有意义的滥用和安全风险"
  • 核心关注：第三方 Prompt Injection--恶意内容劫持用户的 AI 代理（Browser、ChatGPT Agent 等）
  • 攻击面分析：Agentic AI 的攻击面远大于对话式 AI，因为代理可以执行真实操作（浏览网页、调用 API、操作文件）
  • 防御挑战：间接 Prompt Injection 尤其难以防御--恶意指令嵌入在代理处理的外部内容中
深度分析：这一计划的推出时机值得玩味--正值 Agentic AI 产品快速扩张之际。Prompt Injection 在代理场景下的危害远超聊天场景：在聊天中，最坏情况是生成不当内容；在代理场景中，攻击者可能通过被 AI 访问的恶意网页控制代理执行任意操作。OpenAI 通过悬赏的方式发动社区力量来发现这些漏洞，是一种务实的安全策略。
OpenAI 官方公告

GitHub Copilot 数据训练政策：技术与法律影响 政策

GitHub 宣布自 2026 年 4 月 24 日起，默认使用 Copilot Free/Pro/Pro+ 用户的交互数据训练 AI 模型。此变更的技术和商业背景分析：

  • 数据范围：输入（prompt）、输出（completion）、代码片段及关联上下文
  • 豁免对象：Copilot Business 和 Enterprise 用户不受影响
  • 退出机制：用户可通过隐私设置 opt-out，此前已设置的偏好被保留
  • 商业动机：在 Cursor、Claude Code、Windsurf 等竞品压力下，GitHub 需要更多高质量训练数据提升模型
  • 法律风险：可能面临 GDPR（欧盟）和 CCPA（加州）下的数据保护挑战
深度分析：GitHub 的这一决策本质上是 AI 训练数据竞争的缩影。随着公开代码数据被充分利用，各厂商开始争夺"交互数据"--即人类如何使用 AI 编写代码的数据，这类数据对模型微调（fine-tuning）和 RLHF 的价值远高于静态代码库。建议开发者：(1) 审查 Copilot 隐私设置；(2) 对涉及专有算法的代码考虑使用 Copilot Business 或替代工具；(3) 关注后续的法律挑战进展。
